# 国乙同人文 · 表格工坊

> N°024 · 国乙同人项目 — 上传 docx → AI 自动提取六层梗 + 三选项 · 一键导出 xlsx

## 这是什么

一个浏览器端的国乙同人文批量处理工具。把 docx 小说文件拖进去，AI 自动解析出六层梗分析（噱头/世界观/人设/情节/情绪/结局）和三类文游选项（进攻/防御/暧昧），最后导出为标准 Excel 表格。

## 三步工作流

| Step | 卡片 | 做什么 |
|------|------|--------|
| **Step 1** | 大模型 API | 配置 API Base URL、模型、API Key，支持 OpenAI / Anthropic / DeepSeek 等兼容接口，可选本地代理解决 CORS |
| **Step 2** | 拖入 docx | 拖拽或选择 .docx 文件（支持多文件批量），勾选「自动分析」则上传后自动走 AI 流程 |
| **Step 3** | 导出 | 查看已分析的小说队列，一键导出 JSON 备份或 xlsx 表格 |

## 核心功能

### docx 自动解析
- 拖入 .docx 文件，自动清洗正文（去广告、去尾部引导语、去作者注）
- 自动识别游戏归属（恋与深空 / 光与夜之恋 / 未定事件簿 / 时空中的绘旅人 / 恋与制作人 / 世界之外）并匹配角色名
- 提取标题、角色、正文

### AI 六层梗分析
与 AI 协作提取小说的六层内容结构：
1. **噱头层** — 一句话卖点
2. **世界观层** — 故事背景设定
3. **人设层** — 角色关系与特点
4. **情节层** — 核心情节链
5. **情绪层** — 情感体验标签
6. **结局层** — 故事收束方式

支持三种 AI 交互方式：
- **AI 自动** — 一键自动分析
- **生成提示词** — 复制结构化提示词到外部 AI 工具
- **粘贴回复** — 将 AI 回复粘贴回来批量解析

### 文游选项生成
为每篇小说生成三类选项（不是具体动作，而是玩家的不同应对姿态）：
- **A · 进攻** — 玩家主动、大胆、推进关系
- **B · 防御** — 玩家退缩、回避、防御
- **C · 暧昧** — 玩家暧昧、挑逗、试探

### 导出格式
- **JSON** — 完整数据备份，可重新导入
- **Excel (xlsx)** — 符合项目表格规范：A 列世界观+同人+六层梗、D 列引流文案、E 列文游选项；导出时自动去除首行重复标题

## 使用方式

1. 用浏览器打开 `novel_table_generator.html`
2. **Step 1** — 打开「大模型 API」卡片，填入 API 地址、模型名和 Key，点击「测试」确认连通
3. **Step 2** — 将 .docx 文件拖入上传区，或点击手动粘贴正文
4. 在编辑表单中检查 AI 提取结果，微调六层梗和文游选项
5. 点击「添加到表格」加入队列
6. **Step 3** — 在顶部卡片点击「导出 Excel」下载最终表格

> 提示：勾选「自动分析」后，拖入文件会自动走 AI 流程，适合批量处理。

## 技术栈

- 纯前端，零依赖后端
- **mammoth.js** — docx 转 HTML/纯文本
- **SheetJS (xlsx)** — Excel 读写
- 兼容 OpenAI API 格式的大模型接口（DeepSeek、Anthropic 等均可）
- 浮动卡片布局 + 拖拽重排（vanilla JS）
- 所有配置（API Key 等）存储在 localStorage

## 注意事项

- API Key 仅存储在浏览器本地 localStorage，不会上传到任何第三方
- 如遇 CORS 跨域问题，勾选「本地代理」并运行 `node api_proxy.js`
- 建议定期导出 JSON 备份，避免浏览器数据丢失
- 批量处理时建议先测试少量文件，确认 AI 输出格式稳定后再全量跑
